# План реализации: Projects — базовый CRUD

Дизайн: `docs/designs/2026-05-24-projects.md`

---

## Обзор

Реализация полного CRUD проектов: создание, список, просмотр, редактирование, удаление. Текущий пользователь автоматически становится owner. Slug генерируется автоматически из name. Навигация: `/projects` — главная страница приложения.

Prisma-схема уже содержит модель `Project` — миграции не нужны.

**Рекомендуемый размер команды:** 3 агента

---

## Фаза 1 — Бэкенд и фронтенд-сервис (параллельно)

*Без зависимостей между задачами. Все агенты работают параллельно.*

Бэкенд-модуль и фронтенд-сервис не пересекаются по файлам. Установка `slugify` — отдельная задача, потому что `package.json` — общий файл.

### Задача 1.1: Установка slugify + утилита генерации slug

Установить пакет `slugify`, создать утилиту генерации slug с транслитерацией и случайным суффиксом.

- [x] Установить `slugify`: `npm install slugify`
- [x] Создать `apps/api/src/common/utils/generate-slug.ts`:
  - Функция `generateSlug(name: string): string`
  - Алгоритм: `slugify(name)` → lowercase → обрезка до 50 символов → добавление суффикса `-xxxx` (4 символа a-z0-9)
  - Если после slugify строка пустая (name из одних спецсимволов) — выбросить `BadRequestException` с сообщением «Название должно содержать хотя бы одну букву или цифру»
  - Экспортировать функцию

### Задача 1.2: ProjectsModule (бэкенд — DTO, сервис, контроллер)

Реализовать полный бэкенд-модуль проектов: DTO, сервис с бизнес-логикой, контроллер с REST-эндпоинтами.

- [x] Создать `apps/api/src/projects/dto/create-project.dto.ts`:
  - Класс `CreateProjectDto` с class-validator декораторами
  - `name`: `@IsString()`, `@IsNotEmpty()`, `@MinLength(1)`, `@MaxLength(100)`
  - `description`: `@IsOptional()`, `@IsString()`, `@MaxLength(2000)`
- [x] Создать `apps/api/src/projects/dto/update-project.dto.ts`:
  - Класс `UpdateProjectDto` с class-validator декораторами
  - `name`: `@IsOptional()`, `@IsString()`, `@MinLength(1)`, `@MaxLength(100)`
  - `description`: `@IsOptional()`, `@ValidateIf((o) => o.description !== null)`, `@IsString()`, `@MaxLength(2000)` — принимает `string | null` (null очищает поле)
- [x] Создать `apps/api/src/projects/dto/project-response.dto.ts`:
  - Интерфейс `ProjectResponseDto`: `id`, `slug`, `name`, `description: string | null`, `createdAt: string`, `updatedAt: string`
  - Без `ownerId` — пользователь видит только свои
- [x] Создать `apps/api/src/projects/projects.service.ts`:
  - Инъекция `PrismaService` (глобальный, без импорта модуля)
  - `create(userId: string, dto: CreateProjectDto): Promise<ProjectResponseDto>` — генерация slug через `generateSlug()`, retry до 5 раз при коллизии уникальности, `prisma.project.create()`, маппинг через `toResponseDto()`
  - `findAll(userId: string): Promise<ProjectResponseDto[]>` — `prisma.project.findMany({ where: { ownerId: userId }, orderBy: { createdAt: 'desc' } })`, маппинг каждого
  - `findBySlug(userId: string, slug: string): Promise<ProjectResponseDto>` — `findFirst({ where: { slug, ownerId: userId } })`, если не найден → `NotFoundException('Проект не найден')`
  - `update(userId: string, slug: string, dto: UpdateProjectDto): Promise<ProjectResponseDto>` — сначала `findFirst` для проверки ownership, затем `prisma.project.update()` с условным включением полей, маппинг
  - `delete(userId: string, slug: string): Promise<void>` — `findFirst` для проверки ownership, затем `prisma.project.delete()`. Каскадное удаление issues/labels обеспечивает Prisma-схема
  - `private toResponseDto()` — маппинг Prisma-модели в `ProjectResponseDto`, `createdAt`/`updatedAt` через `.toISOString()`
- [x] Создать `apps/api/src/projects/projects.controller.ts`:
  - `@Controller('projects')`
  - `POST /` → `create()`, `@HttpCode(HttpStatus.CREATED)`, `@Body() dto: CreateProjectDto`, `@CurrentUser() user: JwtUser`
  - `GET /` → `findAll()`, `@CurrentUser() user: JwtUser`
  - `GET /:slug` → `findBySlug()`, `@Param('slug') slug: string`, `@CurrentUser() user: JwtUser`
  - `PATCH /:slug` → `update()`, `@Body() dto: UpdateProjectDto`, `@Param('slug') slug: string`, `@CurrentUser() user: JwtUser`
  - `DELETE /:slug` → `delete()`, `@HttpCode(HttpStatus.NO_CONTENT)`, `@Param('slug') slug: string`, `@CurrentUser() user: JwtUser`
- [x] Создать `apps/api/src/projects/projects.module.ts`:
  - Импорт: ничего (PrismaModule глобальный)
  - Controllers: `ProjectsController`
  - Providers: `ProjectsService`
  - Exports: `ProjectsService` (на будущее — Issues будут его использовать)

**Файлы:** `apps/api/src/projects/` (вся директория — новая)

**Скиллы:** `nestjs-best-practices`

**Важно:** Эта задача НЕ редактирует `app.module.ts` и НЕ редактирует `package.json`. Импортирует `generateSlug` из `../common/utils/generate-slug.js`. Если задача 1.1 ещё не завершена — создать локальный плейсхолдер и отметить в сообщении лиду.

## Примечания

- **Prisma-схема не меняется** — модель `Project` уже существует с миграции `init-all-entities`
- **Каскадное удаление** уже настроено в Prisma: `Issue → Project (onDelete: Cascade)`, `Label → Project (onDelete: Cascade)`
- **PrismaModule глобальный** — `ProjectsModule` не нужно его импортировать
- **Сортировка списка**: `orderBy: { createdAt: 'desc' }` — зафиксировано в задаче 1.2
- **Slug не меняется** при обновлении name — зафиксировано в дизайне
